算法竞赛中 Vim 的使用

Posted on 2021-05-16 In OI
简单介绍 Vim 的一些操作……

应 YZ 的要求，作如下关于 Vim 的介绍，后续可能还会补充。

基本篇

什么？你还不会 Vim 的基本操作？去看看 vimtutor 吧。

什么？你不知道 Vim 是什么？看这里。

提高篇

此处假定你已经十分熟悉 Vim 的一些操作（hjkl），主要讲一些提升速度的技巧，请 亲身实践。

光标移动

不会真的有人连点 j 和 k 移动光标吧～

使用 <C-u> 和 <C-d> 移动半屏（<C-u> 表示 CTRL-u）。
使用 <C-b> 和 <C-f> 移动整屏。
使用 <C-y> 和 <C-e> 移动一行。（插入模式下，先按一个 <C-x>）

zt、zz、zb 分别将当前行移至屏幕 顶端、中心、底端。

[数字]G 跳转到第 [数字] 行。
gg、G 分别跳转到文件 开头、结尾。

相对行号是个好东西，配合它和 jk 可以实现 精准跳行。更具体地，输入 :set number 和 :set relativenumber，左侧的行数就会显示成 相对行号。 如果你想跳转到某一行，只要看它和当前行的相对位置，输入这个数字，再按 j/k 就可以啦。想要让这些设置“可持久化”，可以把它们写入“配置文件”（:help vimrc 请，注意去掉 :set 前的冒号）。

W 和 B 可以比 w 和 b 更高效地跳单词。

编辑

插入模式下 <C-o> 可以让你执行一条命令（如 <C-o>O 开启上一行）。

使用 <C-h> 代替 Backspace。
使用 <C-m> 代替回车。
使用 <C-c> 退出编辑模式。

按 <C-v> 可以进入块选中模式，选中一个块，可以实现批量修改（c / s / r）、向前插入（I，又叫插入）、向后插入（A，又叫追加）等操作。 比如用来批量注释。

按 v 进入选中模式，其后可接“范围”，如 当前单词（iw）、引号内的内容（i"）、大中小括号内的内容（i( / i)）等，i 也可换成 a。
类似地，按 c 后面接“范围”，就是更改范围内的内容。cc 或 S 是修改当前行。
y 后面接“范围”，就是复制范围内的内容。
……

其实很多键（如 i, s, a）转换一下大小写就可以实现类似的另一个功能，请读者自行尝试。（然而方向键没有，普通模式下 J 是用来吞回车的）

自动补全

并不那么自动的补全。

输入一个 已出现过 单词的“前缀”，按 <C-p> / <C-n> 就会自动补全啦。
<C-x><C-v> 可以补全 vim 一些指令、选项，在写配置的时候尤其有用。
<C-x><C-l> 可以补全一整行。

详见 :help ins-completion。
